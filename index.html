Sistema de Modos

O formulário opera em dois modos distintos, controlados por um banner sticky abaixo do cabeçalho. O modo é persistido em localStorage e aplicado automaticamente no carregamento.

🟢
Modo Interno (default)
Todas as secções visíveis e editáveis.
— Secção Referência do Processo: visível e editável, marcada com etiqueta "INTERNO".
— Secção A — Identificação do Imóvel: visível e editável, marcada com "VISÍVEL PELO CLIENTE".
— Restante formulário: editável pelo staff interno.
🟡
Modo Cliente
Vista do cliente — enviada/partilhada com o proponente para preenchimento.
— Secção Referência do Processo: completamente oculta (display:none).
— Secção A — Identificação do Imóvel: visível mas bloqueada (readonly + overlay pointer-events:none), marcada com "PREENCHIDO PELA ALYX".
— Restante formulário: editável pelo cliente.
Implementação Técnica
Variável de estadocurrentModo — 'interno' | 'cliente'
Função toggletoggleModo() → chama applyModo()
PersistêncialocalStorage.setItem('alyx_kyc_modo', currentModo)
CSS — secção internabody.cliente-mode .internal-section { display:none !important }
CSS — secção readonlybody.cliente-mode .readonly-section — inputs com pointer-events:none + overlay
Validação no clienteCampos da .readonly-section são excluídos de validateRequired() em Modo Cliente
Etiquetas visuais::before pseudo-elemento nas secções com texto de contexto
ℹPara partilhar com o cliente: mudar para Modo Cliente e partilhar o URL. O modo fica guardado no browser. Em produção, o ideal é gerar um link com parâmetro ?modo=cliente que force o modo automaticamente no carregamento (item no roadmap).
Estrutura do Formulário

10 secções. As etiquetas indicam quem pode ver/editar cada secção. Interno apenas staff   Só leitura visível pelo cliente, preenchido internamente   Cliente preenchido pelo cliente

—
Referência do Processo Interno
Secção unificada — idêntica para Pessoa Singular e Pessoa Coletiva (v2.1). Referência interna gerada automaticamente (ALYX-AAAA-XXXX, persistida em localStorage). Campo Proprietário / Nome do Credor obrigatório — serve ambos os contextos (singular e coletiva). ID Imóvel obrigatório. NIPC opcional. Completamente oculta em Modo Cliente.
A
Identificação do Imóvel Só leitura (cliente)
Morada completa, código postal, localidade, proprietário e NIPC — todos obrigatórios internamente. Em Modo Cliente: visível mas todos os campos bloqueados com overlay de protecção e etiqueta "PREENCHIDO PELA ALYX".
G
Mediação Imobiliária Cliente
Opcional. Denominação social, NIPC, marca, licença AMI, responsável, telefone, email.
B
Proposta Cliente
Propósito do negócio, destino do bem (checkboxes múltiplos + "Outro"), valor, sinal CPCV, prazo escritura, tipo de fundos, % financiamento, banco financiador (condicional), proveniência de fundos próprios, meios de pagamento, IBANs.
C
Proponente Cliente
Toggle Pessoa Singular / Pessoa Coletiva — condiciona as secções C, E (titulares) e documentos obrigatórios.

Singular: dados pessoais, documento (select IRN/AIMA/Outra), NIF, residência, profissão, estado civil, regime matrimonial, contacto. Secção cônjuge condicional (Sim/Não) com todos os campos obrigatórios quando activa.

Coletiva: denominação social, NIPC, CAEs, objeto social, sede, forma jurídica, data/país de constituição, sucursais, responsável de contacto, código RCBE. Secção E dinâmica (Titulares) com blocos adicionáveis por titular — nome, cargo, % capital, beneficiário efetivo, NIF, residência, documento.
D
Representante Legal Cliente
Condicional (Sim/Não). Dados pessoais, documento (select IRN/AIMA/Outra), NIF, residência, contacto. Todos os campos obrigatórios quando activo.
E
Pessoa Politicamente Exposta Cliente
4 questões Sim/Não conforme Lei 83/2017 (art. 2.º, n.º 1, al. cc), w) e dd), e arts. 19.º e 39.º). Ao responder Sim, todos os campos tornam-se obrigatórios. Datas de início/fim com input[type=month]. Definições legais completas em accordion: PeP (13 funções), membros próximos família (4 categorias), pessoas estreitamente associadas (3 categorias), titular outro cargo político/público (24 cargos listados).
F
Documentos Obrigatórios Cliente
Lista diferente conforme tipo de proponente. Singular: doc. identificação, NIF, procuração, fatura serviço público, IBAN. Coletiva: certidão RC + RCBE, docs titulares, NIF, deliberação/procuração, IBAN. Secção comum: comprovativo de origem de fundos (mínimo 1 obrigatório, 6 opções). Formatos aceites: PDF, JPG, PNG. Tamanho máximo: 10MB por ficheiro.
H
Proteção de Dados & Declarações Cliente
Caixa RGPD em accordion expansível com texto legal completo da ALYX Properties ao abrigo do RGPD (UE 2016/679) e Lei 83/2017 — inclui base legal, finalidade, prazo de conservação, direitos do titular e contacto para exercício de direitos (dataprotection@algebracapital.pt). Declarações do(s) Proponente(s) com 4 pontos expressos (condições do imóvel, não indemnização, veracidade dos dados, obrigações AML/CFT). Exportação PDF com pré-visualização. Autenticação CMD e upload de assinatura física para 2 proponentes.

Lógica Condicional

Todos os comportamentos condicionais implementados e a função JS responsável.

Modo Interno → Modo ClientetoggleModo() / applyModo()
Condição: botão no banner clicado
Oculta .internal-section. Bloqueia campos de .readonly-section com readonly e tabindex="-1". Altera labels do banner. Exclui campos readonly de validateRequired(). Estado guardado em localStorage.
Toggle Singular / ColetivasetType(type)
Condição: botão de tipo clicado
Mostra/oculta #section-singular, #section-coletiva, #docs-singular, #docs-coletiva. Aplica/remove data-req nos campos da secção activa. Salva tipo em localStorage. Secção Referência mantém-se inalterada (unificada).
Titular — Adicionar / RemoveraddTitular() / removeTitular(n)
Condição: botão "Adicionar Titular" ou "Remover" clicado
Cria/remove bloco HTML com campos de titular. Em modo coletiva activo, aplica data-req automaticamente aos novos campos. Actualiza progresso.
Banco FinanciadorcheckFinPct()
Condição: % Financiamento > 0%
Campo aparece e torna-se obrigatório. Se 0% ou vazio, esconde e remove data-req.
Cônjuge — SimynToggle()
Condição: botão Sim clicado em questão de cônjuge
Secção #conj_det expande. Todos os campos tornam-se obrigatórios via data-req-pt/en. Ao clicar Não: campos limpos e data-req removido.
Representante Legal — SimynToggle()
Condição: botão Sim clicado
Secção #rep_det expande. Todos os campos (exceto NIF estrangeiro) tornam-se obrigatórios.
Entidade Emitente — OutratoggleIssuerOther()
Condição: select = 'outro'
Campo de texto livre aparece para especificar a entidade. Aplicado em Proponente Singular, Representante Legal e Titulares (Coletiva).
PeP — Questões 1 a 4ynToggle()
Condição: botão Sim clicado em qualquer questão PeP
Campos expandem e tornam-se todos obrigatórios, incluindo início de exercício (input[type=month]). Flag pep_flag activa no PDF e na submissão backend.
Comprovativo de fundos — item marcadofundChkToggle()
Condição: checkbox de comprovativo marcada
Se não há ficheiro carregado, mostra aviso junto ao item. Actualiza validação de documentos.
Destino do Bem / Fundos — OutrotoggleChkOutro()
Condição: checkbox 'Outro' marcada
Campo de texto livre aparece para especificação. Remove-se ao desmarcar.
RGPD — accordiontoggleGdpr()
Condição: clique no cabeçalho da caixa RGPD
Expande/colapsa o corpo com texto legal completo da ALYX Properties (RGPD + Lei 83/2017 + direitos do titular + contacto DPO). Chevron roda 180°.
Definições Legais — accordiontoggleAnnex()
Condição: botão "Ver notas legais" clicado
Expande/colapsa definições legais completas: PeP (13 funções), membros família próximos (4 categorias), pessoas estreitamente associadas (3 categorias), titular outro cargo político ou público (24 cargos conforme Lei 83/2017).
Idioma — PT / ENsetLang(lang)
Condição: botão de idioma clicado
Actualiza todos os elementos com data-i18n, data-req-pt/data-req-en, e labels do banner de modo. Preferência guardada em localStorage.
Tema — Dark / LighttoggleTheme()
Condição: botão de tema clicado no cabeçalho
Adiciona/remove classe light de body. Variáveis CSS comutam entre :root (dark) e body.light (light). Preferência guardada em localStorage.

Endpoints API

Python + FastAPI. Base URL: https://api.alyxproperties.eu/api/v1

POST/kyc/submit

Recebe o formulário KYC completo. Valida server-side, guarda na BD, dispara emails automáticos. Suporta Singular e Coletiva. Campo ref_interna enviado pelo frontend (gerado em localStorage) ou gerado pelo servidor se ausente.

Body (multipart/form-data)
form_dataobject (JSON)Todos os campos do formulário, incluindo todos os titulares (Coletiva)
proposer_typestring'singular' | 'coletiva'
documentsarray<file>Ficheiros carregados — PDF, JPG, PNG. Máx. 10MB cada.
langstring'pt' | 'en'
modostring'interno' | 'cliente' — modo em que foi submetido
ref_internastringALYX-AAAA-XXXX — enviada pelo frontend se já existir em localStorage
pep_flagbooleanTrue se qualquer questão PeP = Sim (calculado no frontend)
Response 200
status'success'
ref'ALYX-2025-0001'
submission_idUUID v4
timestamp'2025-03-24T10:30:00Z'
POST/kyc/upload

Upload individual de documento antes da submissão final. Retorna URL de armazenamento. Aceite antes ou durante a submissão.

Body (multipart/form-data)
filefilePDF, JPG ou PNG. Máximo 10MB.
doc_typestringSingular: 'f1','f3','f4','f5','f7' / Coletiva: 'cc1','cc2','cc3','cc4','cc5' / Fundos: 'g1'–'g6'
ref_internastringReferência para associar o ficheiro (ALYX-AAAA-XXXX)
Response 200
file_idUUID v4
url'https://storage.cloudinary.com/...'
doc_typechave do documento

# Base de dados
DATABASE_URL=postgresql://user:password@localhost:5432/alyx_kyc

# Cloudinary (armazenamento de ficheiros)
CLOUDINARY_CLOUD_NAME=o_teu_cloud_name
CLOUDINARY_API_KEY=a_tua_api_key
CLOUDINARY_API_SECRET=o_teu_api_secret

# SendGrid (emails)
SENDGRID_API_KEY=SG.xxxxxxxxxxxxxxxxxxxx
EMAIL_FROM=kyc@alyxproperties.eu
EMAIL_TEAM=compliance@alyxproperties.eu

# Protecção de dados (DPO)
EMAIL_DPO=dataprotection@algebracapital.pt

# Painel de administração
ADMIN_SECRET_KEY=gerar: python -c "import secrets; print(secrets.token_hex(32))"
ADMIN_EMAIL=admin@alyxproperties.eu

# Aplicação
ENVIRONMENT=development
FRONTEND_URL=http://localhost:3000
ALLOWED_ORIGINS=http://localhost:3000,https://alyxproperties.eu,https://alyx-kyc.vercel.app

# Retenção RGPD
IP_ANONYMIZE_DAYS=30
DATA_RETENTION_YEARS=7
✕Nunca expor estas variáveis no frontend. Em produção usar variáveis de ambiente do servidor (Vercel Environment Variables, Heroku Config Vars, etc.).
ℹEm desenvolvimento, instalar python-dotenv e chamar load_dotenv() no início do main.py.
⚠DATA_RETENTION_YEARS=7 — prazo mínimo de conservação para dados AML/CFT conforme Lei n.º 83/2017, art. 51.º.
⚠EMAIL_DPO — endereço de exercício de direitos RGPD constante da caixa de proteção de dados do formulário. Manter sincronizado com o texto legal.
